# Contributing to StudyFlow

Thanks for your interest in contributing.

## Development Setup

1. Fork the repository and clone your fork.
2. Install dependencies:

```bash
npm install
```

3. Start local development:

```bash
npm run dev
```

## Pull Request Guidelines

- Keep PRs focused and small.
- Add clear description of what changed and why.
- Run checks before opening PR:

```bash
npm run lint
npm run build
```

- If UI behavior changes, include screenshots or short videos.

## Commit Messages

Use concise, imperative commit messages, for example:

- `feat: add configurable timer duration`
- `fix: prevent empty plan submission`
- `docs: update android setup`

## Reporting Issues

When opening an issue, include:

- Environment (OS, Node version)
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots/logs if available
